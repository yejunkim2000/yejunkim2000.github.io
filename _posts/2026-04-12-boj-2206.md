## 문제 정보

- 번호: 2206
- 난이도: 골드 4
- 링크: [https://www.acmicpc.net/problem/2206](https://www.acmicpc.net/problem/2206)

## 문제 요약

N×M 격자에서 (1,1)에서 (N,M)까지 최단 거리로 이동한다.
벽(1)은 이동할 수 없지만, 딱 한 번만 벽을 부수고 지나갈 수 있다.
최단 거리를 구하고, 불가능하면 -1을 출력한다.

---

## 핵심 아이디어

처음에 단순 BFS로 접근하면 틀린다.

일반 BFS는 `visited[r][c]`로 방문 여부를 관리하는데,
이 문제는 **벽을 부쉈는지 여부**에 따라 같은 칸이라도 상태가 달라진다.

예를 들어 (3, 5) 칸을 벽을 부수지 않고 방문한 경우와,
이미 벽을 부수고 방문한 경우는 전혀 다른 상태다.
벽을 아직 안 부쉈다면 앞으로 벽을 한 번 더 부술 수 있지만,
이미 부쉈다면 더 이상 부술 수 없기 때문이다.

따라서 visited 배열을 3차원으로 확장한다.

```
visited[r][c][broken]
- broken = 0 : 아직 벽을 부수지 않은 상태
- broken = 1 : 이미 벽을 부순 상태
```

BFS 큐에 `(r, c, broken)` 세 값을 넣어서 상태를 구분한다.

---

## 풀이 과정

**1단계 - 상태 정의**

큐에 넣을 상태: `(행, 열, 벽을 부쉈는가)`
거리 배열: `dist[r][c][broken]`

**2단계 - 이동 처리**

상하좌우로 인접한 칸 `(nr, nc)`를 탐색할 때:

- `map[nr][nc] == 0` (빈 칸) → 그냥 이동. broken 상태 유지
- `map[nr][nc] == 1` (벽) → `broken == 0`일 때만 부수고 이동 가능. broken을 1로 변경

**3단계 - 방문 체크**

`visited[nr][nc][next_broken]`이 false일 때만 큐에 추가.

**4단계 - 도착 확인**

`(N-1, M-1)` 칸에 처음 도달하는 순간의 거리가 최단 거리.

---

## 예제 추적

```
6 4
0100
1110
1000
0000
0111
0000
```

(0,0)에서 출발. 오른쪽으로 가면 벽(0,1)에 막힌다.
벽을 부수고 (0,1)을 지나가면 broken=1이 된다.
이후에는 벽을 만나도 더 이상 부술 수 없다.
이런 식으로 broken=0/1 두 레이어를 BFS로 탐색해서 최단 경로 15를 찾는다.

---

## 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int N, M;
int map[1000][1000];
int dist[1000][1000][2];
int dr[] = {-1, 1, 0, 0};
int dc[] = {0, 0, -1, 1};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> N >> M;

    for (int i = 0; i < N; i++) {
        string s;
        cin >> s;
        for (int j = 0; j < M; j++) {
            map[i][j] = s[j] - '0';
        }
    }

    memset(dist, -1, sizeof(dist));

    queue<tuple<int, int, int>> q;
    q.push({0, 0, 0});
    dist[0][0][0] = 1;

    while (!q.empty()) {
        auto [r, c, broken] = q.front();
        q.pop();

        if (r == N - 1 && c == M - 1) {
            cout << dist[r][c][broken] << '\n';
            return 0;
        }

        for (int d = 0; d < 4; d++) {
            int nr = r + dr[d];
            int nc = c + dc[d];

            if (nr < 0 || nr >= N || nc < 0 || nc >= M) continue;

            if (map[nr][nc] == 0 && dist[nr][nc][broken] == -1) {
                dist[nr][nc][broken] = dist[r][c][broken] + 1;
                q.push({nr, nc, broken});
            }

            if (map[nr][nc] == 1 && broken == 0 && dist[nr][nc][1] == -1) {
                dist[nr][nc][1] = dist[r][c][broken] + 1;
                q.push({nr, nc, 1});
            }
        }
    }

    cout << -1 << '\n';
    return 0;
}
```

## 틀렸던 포인트 / 주의사항

- `visited[r][c]` 2차원 배열로만 관리하면 WA가 난다. 반드시 3차원으로 확장해야 한다.
- 입력을 숫자 사이에 공백 없이 붙여서 주므로 string으로 한 줄씩 읽어야 한다.
- 메모리가 N, M이 최대 1000이므로 `dist[1000][1000][2]`는 약 8MB. 문제 없음.

---

## 배운 점

BFS에서 단순 좌표만으로 상태를 표현하면 안 되는 경우가 있다.
이 문제처럼 **추가 조건(벽을 부쉈는가)이 상태에 영향을 미칠 때**는
visited 배열에 그 조건을 차원으로 추가해서 상태를 분리해야 한다.
이 아이디어는 다양한 BFS 응용 문제에서 반복적으로 등장하는 핵심 패턴이다.
