## 문제 정보

- 번호: 4179
- 난이도: 골드 4
- 링크: [https://www.acmicpc.net/problem/4179](https://www.acmicpc.net/problem/4179)

## 문제 요약

지훈이는 미로에서 탈출해야 한다. 미로 안에는 불이 번지고 있다.
지훈이는 불이 있는 칸으로 이동할 수 없고, 불은 매 초 상하좌우로 번진다.
지훈이가 미로 밖으로 탈출할 수 있는 최단 시간을 구한다. 불가능하면 IMPOSSIBLE.

---

## 핵심 아이디어

**BFS를 두 번 돌린다.**

1. 불 BFS: 각 칸에 불이 언제 도달하는지 미리 계산
2. 지훈이 BFS: 이동할 때 현재 시간이 해당 칸의 불 도달 시간보다 작은 경우만 이동

핵심은 지훈이가 어떤 칸 (r, c)에 시간 t에 도착했을 때,
`fire[r][c] > t` 이어야 이동 가능하다는 것이다.
(불이 도달하지 않았거나, 지훈이가 먼저 지나갔어야 한다)

탈출 조건: 지훈이가 격자의 경계 밖으로 나가면 탈출 성공.

---

## 풀이 과정

**1단계 - 불 BFS**

불이 있는 모든 칸('F')을 큐에 넣고 동시에 BFS를 시작한다.
`fire[r][c]`에 불이 해당 칸에 도달하는 최소 시간을 저장한다.
불이 여러 곳에서 동시에 시작되는 **멀티소스 BFS**다.

**2단계 - 지훈이 BFS**

지훈이 시작 위치('J')에서 BFS 시작.
상하좌우로 이동할 때:
- 격자 밖으로 나가면 → 탈출 성공, 현재 시간 + 1 반환
- `fire[nr][nc] <= dist[r][c] + 1` 이면 → 불이 먼저 와있으니 이동 불가
- 그 외 → 이동 가능

**3단계 - 결과 출력**

BFS가 끝날 때까지 탈출 못 하면 IMPOSSIBLE.

---

## 예제 추적

```
4 4
####
#JF.#  (실제 예제와 다를 수 있음)
#...#
####
```

불 BFS로 각 칸에 불 도달 시간을 먼저 계산.
지훈이 BFS로 불보다 먼저 도달 가능한 경로만 탐색.
경계에 도달하면 탈출 시간 출력.

---

## 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int R, C;
char grid[1001][1001];
int fire[1001][1001];
int dist[1001][1001];
int dr[] = {-1, 1, 0, 0};
int dc[] = {0, 0, -1, 1};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> R >> C;

    queue<pair<int,int>> fq, jq;
    memset(fire, -1, sizeof(fire));
    memset(dist, -1, sizeof(dist));

    for (int i = 0; i < R; i++) {
        string s;
        cin >> s;
        for (int j = 0; j < C; j++) {
            grid[i][j] = s[j];
            if (s[j] == 'F') {
                fq.push({i, j});
                fire[i][j] = 0;
            }
            if (s[j] == 'J') {
                jq.push({i, j});
                dist[i][j] = 0;
            }
        }
    }

    while (!fq.empty()) {
        auto [r, c] = fq.front();
        fq.pop();
        for (int d = 0; d < 4; d++) {
            int nr = r + dr[d];
            int nc = c + dc[d];
            if (nr < 0 || nr >= R || nc < 0 || nc >= C) continue;
            if (grid[nr][nc] == '#') continue;
            if (fire[nr][nc] != -1) continue;
            fire[nr][nc] = fire[r][c] + 1;
            fq.push({nr, nc});
        }
    }

    while (!jq.empty()) {
        auto [r, c] = jq.front();
        jq.pop();
        for (int d = 0; d < 4; d++) {
            int nr = r + dr[d];
            int nc = c + dc[d];
            if (nr < 0 || nr >= R || nc < 0 || nc >= C) {
                cout << dist[r][c] + 1 << '\n';
                return 0;
            }
            if (grid[nr][nc] == '#') continue;
            if (dist[nr][nc] != -1) continue;
            if (fire[nr][nc] != -1 && fire[nr][nc] <= dist[r][c] + 1) continue;
            dist[nr][nc] = dist[r][c] + 1;
            jq.push({nr, nc});
        }
    }

    cout << "IMPOSSIBLE\n";
    return 0;
}
```

## 틀렸던 포인트 / 주의사항

- 불 BFS를 먼저 돌려서 각 칸의 불 도달 시간을 구해야 한다. 지훈이 BFS와 동시에 처리하려 하면 복잡해진다.
- 불이 여러 곳에서 시작되므로 멀티소스 BFS를 써야 한다.
- 격자 밖으로 나가는 순간이 탈출이다. `nr < 0 || nr >= R || nc < 0 || nc >= C` 조건에서 바로 탈출 처리.
- 불 도달 시간이 -1(불이 안 오는 칸)인 경우와 비교할 때 주의해야 한다.

---

## 배운 점

여러 출발점에서 동시에 BFS를 시작하는 **멀티소스 BFS** 패턴을 익혔다.
두 개의 BFS를 순서대로 실행해서 결과를 조합하는 방식도 중요한 테크닉이다.
불과 사람이 동시에 움직이는 문제는 불을 먼저 전처리한 뒤 사람을 탐색하는 게 깔끔하다.
