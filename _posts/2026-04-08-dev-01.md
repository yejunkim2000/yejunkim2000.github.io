## 솔직한 현재 상태

PS(Problem Solving)는 꽤 해왔다.
알고리즘 문제를 풀고, 시간복잡도를 고민하고,
코드가 돌아가는 것에 익숙해졌다.

그런데 막상 "뭔가를 만들어봐" 라고 하면 아무것도 못 한다.
입력을 받고 출력을 내는 것 말고는 해본 게 없다.

## PS와 개발의 차이

PS에서는 문제가 주어진다.
입력 형식이 있고, 출력 형식이 있고, 정답이 있다.

개발은 다르다.
무엇을 만들지 스스로 정해야 하고,
파일을 어떻게 구성할지, 어떤 라이브러리를 쓸지,
에러가 나면 어디서 나는지 전부 혼자 해결해야 한다.

## 부족한 점

- 프로젝트 구조를 어떻게 잡아야 하는지 모름
- 라이브러리, 프레임워크 사용 경험 없음
- Git을 코드 저장 용도 외로 써본 적 없음
- 내가 만든 코드를 남이 쓸 수 있게 다듬는 능력 없음
- 디버깅을 print 말고 다른 방법으로 해본 적 없음

## 왜 이제 시작하는가

CTF나 보안 도구들을 보면 결국 개발 능력이 바탕이 된다.
자동화 스크립트, 익스플로잇 코드, 분석 도구 —
전부 뭔가를 직접 만들 수 있어야 한다.

PS로 쌓은 로직은 있으니까,
이제 그걸 실제로 동작하는 무언가로 만드는 연습을 시작할 때다.

## 앞으로 할 것

PS 예전에 풀었던 걸들을 다시 정리 해볼것이다.
그리고 일단 작은 것부터 만들어본다.
거창한 프로젝트보다 내가 실제로 쓸 수 있는
작은 도구 하나를 완성하는 게 목표다.
