## 문제 정보

- 번호: 9019
- 난이도: 골드 4
- 링크: [https://www.acmicpc.net/problem/9019](https://www.acmicpc.net/problem/9019)

## 문제 요약

4자리 숫자(0~9999)에 4가지 명령어를 적용해서 A를 B로 만드는 최소 명령어 시퀀스를 구한다.

- D: 숫자를 2배. 10000 이상이면 10000으로 나눈 나머지
- S: 숫자에서 1을 뺌. 0이면 9999
- L: 왼쪽으로 한 자리 회전 (1234 → 2341)
- R: 오른쪽으로 한 자리 회전 (1234 → 4123)

---

## 핵심 아이디어

0~9999 사이의 숫자를 노드로, 4가지 명령어를 간선으로 보는 **BFS 최단 경로** 문제다.

각 상태(숫자)는 한 번만 방문하면 되므로 `visited[10000]` 배열로 방문 체크.
경로를 추적하기 위해 `from[10000]`(이전 숫자)와 `cmd[10000]`(사용한 명령어) 배열을 관리한다.

목표 B에 도달하면 역추적으로 명령어 시퀀스를 복원한다.

---

## 풀이 과정

**1단계 - BFS 설정**

시작 상태 A를 큐에 넣고 visited[A] = true.

**2단계 - 4가지 명령어 적용**

현재 숫자 cur에서 D, S, L, R을 각각 적용해 다음 상태를 계산한다.

- D: `(cur * 2) % 10000`
- S: `cur == 0 ? 9999 : cur - 1`
- L: `(cur % 1000) * 10 + cur / 1000`
- R: `(cur % 10) * 1000 + cur / 10`

**3단계 - 경로 추적**

`from[next] = cur`, `cmd[next] = 명령어` 저장.
B에 도달하면 B에서 A까지 역추적해서 명령어를 뒤집어 출력.

---

## L, R 연산 상세

L 연산 (왼쪽 회전):
```
1234 → 2341
cur = 1234
cur % 1000 = 234  (앞 세 자리)
cur / 1000 = 1    (맨 앞 자리)
(234) * 10 + 1 = 2341
```

R 연산 (오른쪽 회전):
```
1234 → 4123
cur = 1234
cur % 10 = 4     (맨 뒷 자리)
cur / 10 = 123   (뒤 세 자리)
4 * 1000 + 123 = 4123
```

## 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int from[10000];
char cmd[10000];
bool visited[10000];

string solve(int A, int B) {
    memset(visited, false, sizeof(visited));
    queue<int> q;
    q.push(A);
    visited[A] = true;
    from[A] = -1;

    while (!q.empty()) {
        int cur = q.front();
        q.pop();

        if (cur == B) {
            string result = "";
            int x = B;
            while (from[x] != -1) {
                result += cmd[x];
                x = from[x];
            }
            reverse(result.begin(), result.end());
            return result;
        }

        int next;

        next = (cur * 2) % 10000;
        if (!visited[next]) {
            visited[next] = true;
            from[next] = cur;
            cmd[next] = 'D';
            q.push(next);
        }

        next = (cur == 0) ? 9999 : cur - 1;
        if (!visited[next]) {
            visited[next] = true;
            from[next] = cur;
            cmd[next] = 'S';
            q.push(next);
        }

        next = (cur % 1000) * 10 + cur / 1000;
        if (!visited[next]) {
            visited[next] = true;
            from[next] = cur;
            cmd[next] = 'L';
            q.push(next);
        }

        next = (cur % 10) * 1000 + cur / 10;
        if (!visited[next]) {
            visited[next] = true;
            from[next] = cur;
            cmd[next] = 'R';
            q.push(next);
        }
    }
    return "";
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int T;
    cin >> T;
    while (T--) {
        int A, B;
        cin >> A >> B;
        cout << solve(A, B) << '\n';
    }
    return 0;
}
```

---

## 틀렸던 포인트 / 주의사항

- A == B인 경우 빈 문자열을 출력해야 한다. `from[A] = -1`로 초기화하면 역추적 시 처리된다.
- L, R 연산에서 자릿수 계산을 잘못하면 틀린다. 위의 계산식을 정확히 구현해야 한다.
- 테스트케이스가 여러 개이므로 매번 visited를 초기화해야 한다.

---

## 배운 점

BFS를 단순 최단 거리가 아닌 **최단 경로(명령어 시퀀스) 추적**에 활용하는 방법을 익혔다.
`from[]`과 `cmd[]` 배열로 역추적하는 패턴은 경로 복원이 필요한 BFS 문제에서 공통으로 사용된다.
0~9999라는 작은 상태 공간 덕분에 BFS로 완전 탐색이 가능하다.
