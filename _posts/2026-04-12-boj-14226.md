## 문제 정보

- 번호: 14226
- 난이도: 골드 4
- 링크: [https://www.acmicpc.net/problem/14226](https://www.acmicpc.net/problem/14226)

## 문제 요약

화면에 이모티콘이 1개 있다. 3가지 연산을 사용해서 화면에 이모티콘을 S개 만드는 최소 시간을 구한다.

- 복사: 화면의 이모티콘을 모두 클립보드에 복사 (1초)
- 붙여넣기: 클립보드의 이모티콘을 화면에 붙여넣기 (1초)
- 삭제: 화면의 이모티콘 하나 삭제 (1초)

---

## 핵심 아이디어

상태를 **(화면의 이모티콘 수, 클립보드의 이모티콘 수)** 쌍으로 정의한다.

처음에 `(화면 수)`만으로 상태를 잡으면 틀린다.
클립보드에 얼마나 있는지에 따라 같은 화면 수에서도 다음 상태가 달라지기 때문이다.

예를 들어 화면에 3개 있고 클립보드에 2개 있는 경우와,
화면에 3개 있고 클립보드에 3개 있는 경우는 완전히 다른 상태다.

따라서 `visited[screen][clipboard]` 2차원 배열로 방문 체크하고
BFS로 최단 시간을 구한다.

S는 최대 1000이므로 screen과 clipboard의 범위를 0~1000으로 제한한다.

---

## 풀이 과정

**1단계 - 상태 정의**

`(screen, clipboard)` → 현재까지 걸린 시간
시작 상태: `(1, 0)`, 시간 0

**2단계 - 3가지 연산 처리**

현재 상태 `(s, c)`에서:

1. 복사: `(s, s)` → 클립보드를 화면과 동일하게 (s가 0이면 의미 없으므로 s > 0 조건)
2. 붙여넣기: `(s + c, c)` → 화면에 클립보드 추가 (c > 0 조건)
3. 삭제: `(s - 1, c)` → 화면 하나 삭제 (s > 0 조건)

**3단계 - 범위 제한**

화면과 클립보드는 0~1000 범위만 다룬다.
S보다 많아지면 삭제로만 접근 가능하므로 1000을 넘을 필요가 없다.

**4단계 - 목표 도달**

`screen == S`인 상태에 처음 도달하면 그 시간이 정답.

## 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int S;
    cin >> S;

    int dist[1001][1001];
    memset(dist, -1, sizeof(dist));

    queue<pair<int,int>> q;
    q.push({1, 0});
    dist[1][0] = 0;

    while (!q.empty()) {
        auto [s, c] = q.front();
        q.pop();

        if (s == S) {
            cout << dist[s][c] << '\n';
            return 0;
        }

        if (s > 0 && dist[s][s] == -1) {
            dist[s][s] = dist[s][c] + 1;
            q.push({s, s});
        }

        if (c > 0 && s + c <= 1000 && dist[s + c][c] == -1) {
            dist[s + c][c] = dist[s][c] + 1;
            q.push({s + c, c});
        }

        if (s > 0 && dist[s - 1][c] == -1) {
            dist[s - 1][c] = dist[s][c] + 1;
            q.push({s - 1, c});
        }
    }

    return 0;
}
```

## 틀렸던 포인트 / 주의사항

- 상태를 화면 수만으로 잡으면 WA. 반드시 `(screen, clipboard)` 쌍으로 관리해야 한다.
- 붙여넣기할 때 `s + c`가 1000을 초과하지 않도록 범위를 제한해야 한다.
- 복사 연산 시 `dist[s][s]`를 갱신하는데, 이때 현재 c 값과 무관하게 클립보드가 s로 바뀐다.

## 배운 점

BFS에서 **상태를 어떻게 정의하느냐**가 풀이의 핵심이다.
직관적으로 보이는 상태 정의가 부족할 때, 추가 정보를 상태에 포함시켜야 한다.
이 문제에서는 클립보드 정보를 상태에 추가함으로써 올바른 BFS가 가능해진다.
2206번(벽 부수고 이동하기)과 같은 **상태 확장** 패턴이다.
