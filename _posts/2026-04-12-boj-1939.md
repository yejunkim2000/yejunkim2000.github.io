## 문제 정보

- 번호: 1939
- 난이도: 골드 4
- 링크: [https://www.acmicpc.net/problem/1939](https://www.acmicpc.net/problem/1939)

## 문제 요약

N개의 섬과 M개의 다리가 있다. 각 다리는 최대 중량 제한이 있다.
공장 A에서 공장 B로 한 번에 옮길 수 있는 물건의 최대 중량을 구한다.
경로상 가장 약한 다리가 그 경로의 한계를 결정한다.

---

## 핵심 아이디어

경로의 최소 간선 중 최대값을 구하는 문제다.

**이분탐색 + BFS** 조합으로 푼다.

`mid`값을 기준으로 중량 제한이 `mid` 이상인 다리만 사용할 수 있을 때,
A에서 B까지 도달 가능한지 BFS로 확인한다.

- 가능하다 → 더 무거운 중량도 가능할 수 있으니 `lo = mid + 1`
- 불가능하다 → 중량을 줄여야 하니 `hi = mid - 1`

이분탐색 범위는 다리 중량의 최솟값 ~ 최댓값.

---

## 풀이 과정

**1단계 - 이분탐색 범위 설정**

모든 다리의 중량을 수집하고 lo=1, hi=최대중량으로 설정한다.

**2단계 - BFS 함수 작성**

특정 중량 `limit`이 주어졌을 때,
중량 제한 >= limit인 다리만 사용해서 A → B 도달 가능한지 BFS로 확인한다.

**3단계 - 이분탐색 실행**

`bfs(mid)`가 true이면 `lo = mid + 1`, false이면 `hi = mid - 1`.
루프가 끝나면 `hi`가 정답이다.

---

## 예제 추적

```
3 3
1 2 2
3 1 3
2 3 2
```

다리 중량: 2, 3, 2. 이분탐색 범위 [1, 3].

mid=2: 중량 >= 2인 다리만 사용. 1→2(2), 2→3(2) → 1에서 3 도달 가능 → lo=3
mid=3: 중량 >= 3인 다리만 사용. 1→3(3)만 존재 → 1에서 3 도달 가능 → lo=4
lo > hi → 종료. 정답 = hi = 3.

## 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int N, M;
vector<pair<int, int>> graph[100001];
int A, B;

bool bfs(int limit) {
    vector<bool> visited(N + 1, false);
    queue<int> q;
    q.push(A);
    visited[A] = true;

    while (!q.empty()) {
        int cur = q.front();
        q.pop();

        if (cur == B) return true;

        for (auto [next, weight] : graph[cur]) {
            if (!visited[next] && weight >= limit) {
                visited[next] = true;
                q.push(next);
            }
        }
    }
    return false;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    cin >> N >> M;

    int maxWeight = 0;
    for (int i = 0; i < M; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        graph[u].push_back({v, w});
        graph[v].push_back({u, w});
        maxWeight = max(maxWeight, w);
    }

    cin >> A >> B;

    int lo = 1, hi = maxWeight, ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (bfs(mid)) {
            ans = mid;
            lo = mid + 1;
        } else {
            hi = mid - 1;
        }
    }

    cout << ans << '\n';
    return 0;
}
```

## 틀렸던 포인트 / 주의사항

- 단순 BFS만으로는 최적 중량을 구하기 어렵다. 이분탐색과 조합해야 한다.
- 다리는 양방향이므로 양쪽으로 간선을 추가해야 한다.
- lo, hi 업데이트 방향에 주의. 가능할 때 ans를 갱신하고 lo를 올린다.

---

## 배운 점

최적값을 직접 구하기 어려울 때 **이분탐색으로 결정 문제로 바꾸는 전략**이 핵심이다.
"최대 중량이 X일 때 가능한가?" 라는 결정 문제는 BFS로 쉽게 풀린다.
이 패턴(이분탐색 + 그래프 탐색)은 최솟값의 최댓값, 최댓값의 최솟값 유형에서 자주 등장한다.
