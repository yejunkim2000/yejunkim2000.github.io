## 문제

유이는 길이 $N$의 수열 $A$를 오름차순 정렬하려 한다. 그러나 평범한 정렬 알고리즘은 너무 뻔하다고 생각한 유이는 다음 두 가지 연산 중 하나만을 사용하여 수열을 정렬하려 한다.


	연산 $1$: $A_i, A_{i+1}$의 합이 홀수인 $i$를 골라 이 두 원소를 교환한다. $(1\leq i
	연산 $2$: $A_i, A_{i+1}$의 합이 짝수인 $i$를 골라 이 두 원소를 교환한다. $(1\leq i


주어진 수열 $A$에 대해, 두 가지 연산 중 한 가지 연산만을 $0$번 이상 사용해서 오름차순 정렬이 가능하다면 유이는 자신이 운이 좋다고 생각하고, 불가능하다면 운이 나쁘다고 생각한다.

유이는 지금 자신이 얼마나 운이 좋은지 알고 싶다. 당신이 대신 확인해주자.

## 입력

첫째 줄에 $N$이 주어진다.

둘째 줄에 $A_1,A_2,\cdots ,A_N$이 공백으로 구분되어 주어진다.

## 출력

첫째 줄에 연산 $1$만을 사용했을 때 유이의 운이 좋다면 So Lucky를, 운이 나쁘다면 Unlucky를 출력한다.

둘째 줄에 연산 $2$만을 사용했을 때 유이의 운이 좋다면 So Lucky를, 운이 나쁘다면 Unlucky를 출력한다.

## 풀이

```cpp
#include <iostream>
#include <vector>
#include <string>
using namespace std;

int N;
vector<int> A, temp;

bool canSwap(int a, int b, int type) {
    int sum = a + b;
    if (type == 1) return (sum % 2 == 1);
    else return (sum % 2 == 0);
}

bool mergeSort(int l, int r, int type) {
    if (r - l <= 1) return true;

    int m = (l + r) / 2;
    if (!mergeSort(l, m, type)) return false;
    if (!mergeSort(m, r, type)) return false;

    int i = l, j = m, idx = l;
    while (i < m && j < r) {
        if (A[i] < A[j]) temp[idx++] = A[i++];
        else if (A[i] > A[j]) {
            if (!canSwap(A[i], A[j], type)) return false;
            temp[idx++] = A[j++];
        } else temp[idx++] = A[i++];
    }
    while (i < m) temp[idx++] = A[i++];
    while (j < r) temp[idx++] = A[j++];

    for (int k = l; k < r; ++k) A[k] = temp[k];
    return true;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> N;
    A.resize(N);
    temp.resize(N);
    for (int i = 0; i < N; ++i) cin >> A[i];

    vector<int> backup = A;

    if (mergeSort(0, N, 1)) cout << "So Lucky\n";
    else cout << "Unlucky\n";

    A = backup;

    if (mergeSort(0, N, 2)) cout << "So Lucky\n";
    else cout << "Unlucky\n";

    return 0;
}


```

## 해설

### 핵심 아이디어
운이 좋은(So Lucky) 배열을 정렬 + 홀짝성(parity) 분석으로 판별.

### 풀이 과정
- 정렬 후 인접 원소 간 홀짝성 일치 여부로 특정 조건 판별.
- 애드 혹(Ad-hoc): 규칙을 직접 관찰하여 구현.

### 시간복잡도
O(N log N)
